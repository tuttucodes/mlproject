# 🧠 Brain Tumor Segmentation with 3D Viewer

## ML Project by Rahul & Krishnaa for Dr. Valarmathi | VIT Chennai

End-to-end brain tumor segmentation system using BraTS 2024 dataset with **7 novel ML features**, a complete training pipeline, and an interactive 3D viewer dashboard.

---

## Quick Start

```bash
# 1. Clone and setup
git clone https://github.com/your-org/ml-brain-tumor.git
cd ml-brain-tumor
chmod +x setup.sh && ./setup.sh

# 2. Start backend
source venv/bin/activate
cd backend && uvicorn app.main:app --reload --port 8000

# 3. Open API docs
open http://localhost:8000/docs
```

**Or with Docker:**
```bash
docker compose up --build
```

## 7 Novel ML Features

| # | Feature | Endpoint | Description |
|---|---------|----------|-------------|
| 1 | **Tumor Grading** | `POST /api/grade-tumor` | WHO Grade I-IV classification from radiomic features |
| 2 | **Uncertainty Quantification** | `POST /api/uncertainty` | MC-Dropout (20 iterations) → voxel-wise entropy map |
| 3 | **Comparative Analysis** | `POST /api/compare` | Dice, Jaccard, Hausdorff between two segmentations |
| 4 | **Radiomics Extraction** | `POST /api/radiomics` | 63+ radiomic features (shape, first-order, GLCM, GLRLM) |
| 5 | **Quality Assurance** | `POST /api/qa` | 8 automated QA checks with quality score |
| 6 | **Survival Prediction** | `POST /api/survival-prediction` | Overall survival estimation with Kaplan-Meier curve |
| 7 | **Attention Maps** | `POST /api/attention-maps` | Grad-CAM explainability maps |

---

## Dataset: BraTS 2024

- **Source**: Kaggle (`brats2024-small-dataset`) / Synapse (`syn53708249`)
- **Modalities**: T1, T1CE, T2, T2-FLAIR
- **Labels**: 0 (background), 1 (necrotic), 2 (edema), 4 (enhancing)
- **Cases**: 1,621 training (GLI subset)
- **Format**: NIfTI (.nii.gz)

The training notebook includes **synthetic data generation** so you can run the full pipeline without downloading the dataset.

## Training (Google Colab)

1. Upload `notebooks/training.ipynb` to Google Colab
2. Select GPU runtime (T4 recommended)
3. Run all cells — the notebook handles:
   - Dataset download (Kaggle) or synthetic data generation
   - 3D U-Net training with Dice + CrossEntropy loss
   - Validation with BraTS metrics (WT/TC/ET Dice)
   - XGBoost grading classifier training
   - MC-Dropout uncertainty demo
   - Model export (.pth, .onnx, .pkl)
   - Google Drive saving

Copy exported weights to `models/` directory for backend deployment.

## API Reference

### Core Segmentation
```bash
curl -X POST http://localhost:8000/api/segment \
  -F "t1=@brain_t1.nii.gz" \
  -F "t1ce=@brain_t1ce.nii.gz" \
  -F "t2=@brain_t2.nii.gz" \
  -F "flair=@brain_flair.nii.gz"
```

### Tumor Grading
```bash
curl -X POST http://localhost:8000/api/grade-tumor \
  -F "seg_file=@segmentation.nii.gz"
```

### Quality Assurance
```bash
curl -X POST http://localhost:8000/api/qa \
  -F "seg_file=@segmentation.nii.gz"
```

Full interactive docs at `http://localhost:8000/docs` (Swagger UI).
